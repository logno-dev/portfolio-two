The Book

As my guide, I have chosen to follow the official Rust language book.

This book is intended for those who have programmed in other languages, but has no requirements for knowledge in a specific language. It can be read chronologically or used as a reference.

The book starts with the installation process and then drops you straight into a project to help build familiarity with the syntax. After it works through a small guessing game program, it then dives into the specific components that make up the Rust programming language.

I will highlight and summarize what I have learned through each chapter as I progress.

The Set-up

With JavaScript, you can open any text editor and start chugging out code and see its effects in the browser. Unfortunately Rust will not work this way. We need to set up our development environment.

Installing Rust

For this set up I will be working on a linux system, and I'll be using Neovim as my text editor.

In order to write in Rust, we must install Rust and a Rust package manager. From what I understand there are a couple of options for this, but I personally simply installed rustup which handles Rust versions as well as installs the rust package manager called cargo. I am on an arch based distro and I use yay as my system package manager, so I can simply run the following: 

# ~/

$ yay -S rustup

Great, now we can create a directory for our project and create our first program. 

# ~/

$ cargo new helloworld

The above command creates a directory within your current directory and initiates a new project including a git repository. Within this project directory, the file /src/main.rs will contain the following code: 

// helloworld/src/main.rs

fn main() {
  println!("Hello, World!");
}

So we already have our first program. Starting from our home directory, we can run the follwing commands to see our first program in action. 

# ~/

$ cd helloworld
$ cargo run
    Compiling helloworld v0.1.0 (/home/logno/helloworld)
      Finished dev [unoptimized + debuginfo] target(s) in 0.44s
        Running `target/debug/helloworld`
  Hello, world!

Great! We are up and running. Now lets learn some basics.

Common Programming Concepts

After the book walks through a simple example program, chapter 3 takes us into the foundations of programming in Rust.

Variables and Mutability

Variables in Rust are immutable by default, which means that once you declare a variable it cannot be changed throughout its life-cycle. At the time of initiation, you may designate a variable as mutable by using the mut key word. Variables are initiated by using the let key word.

Constants are initiated with const and are also immutable. Constants differ from immutable variables in a few ways. Constants may never be mutable. Constants must always declare a type, where variables are allowed to use inferred types. Lastly, constants can be declared in any scope including the global scope and they will be valid through the entire run time of the program. 

// Example variable and constant declarations

  let x = 5;

  let mut y = 3;

  x = 7; // invalid; x is immutable, this will result in a compile time error

  y = 9; // valid; y is mutable

  // this is called shadowing, this creates a new variable named x and assigns it the value of the old x + 1
  let x = x + 1; // valid

  {
    let x = x * 2; // this is also valid but will the value of x will return to 6 at the end of this scope.
  }

  const HOURS_IN_SHIFT: u32 = 8; // this is a constant, constants use the SCREAMING_SNAKE case as their naming convention

Data Types

As I have mentioned previously, Rust being a statically typed language was one of the main reasons I gravitated to it. If a value is unambiguous, the compiler can infer the type, but if there is multiple possible types, a type must be declared. At this point in the book, only a few types are covered.

Scalar Types

Scalar types are single values. The four primary scalar types in rust include: integers, floating-point numbers, Booleans, and characters.

Integer: Numbers without fractional components. Integers can be declared as 8, 16, 32, 64, or 128-bit which determines the amount of space the value takes in memory. A u and i is used to denote if the integer is unsigned or signed respectively, where a signed integer can span the negative number range and an unsigned may only be positive integers. An additional integer type of isize and usize can be used to adjust to the architecture of the computer the program is running on.

Floating-point type: Numbers that contain a fractional component. Rust has two floating-point types; f32 and f64. Rust defaults to 64-bit because it is roughly the same speed as the 32-bit variant, but is capable of greater precision.

Boolean: As in most languages, boolean types are represented as true and false. Booleans are specified using bool

Character: coming from JavaScript, this type seemed a bit odd to me, but I suppose I will learn where this will be useful. The character type represents, a single character, which could be a letter, or a unicode value. This type is specified using single quotes as opposed to double quotes when declaring string literals.
